# Дизайн: визуальное усиление weekly IT executive report

Дата: 2026-04-20
Репозиторий: `/tmp/west-project`
Целевой артефакт: `20-04-2026.html`

## 1. Цель изменения

Обновить уже опубликованный HTML-отчет так, чтобы он лучше работал как страница для встречи с руководителями:

- меньше текста;
- больше метрик и графиков;
- отдельный заметный блок по `ABC-классификации клиентов в 1С`;
- компактный визуальный Гант по складскому проекту;
- более явная выжимка ключевых результатов недели из отчета `13–17 апреля 2026`.

Это не новый отчет с нуля. Это точечный визуальный редизайн существующей страницы без смены базового стиля и без увеличения ее сложности.

## 2. Новые акценты страницы

После правки страница должна явно показывать три ключевые темы недели:

- `ABC-клиенты в 1С`;
- `Склад`;
- `Сайт`.

При этом `Склад` и `Сайт` остаются проектными блоками, а `ABC` становится отдельным бизнес-результатом недели, а не просто строкой в общем тексте.

## 3. Источники данных для обновления

Обязательные источники:

- `Отчёт_IT_неделя_13-17_апреля.docx`;
- `Описание_ABCКлиентов.docx`;
- `Диаграмма_Ганта (1).xlsx`;
- текущий HTML `20-04-2026.html`;
- уже использованные метрики по инцидентам;
- Google Sheet по задачам сайта.

## 4. Что должно измениться в содержании

### 4.1 Блок `Обзор`

Верх страницы должен стать более числовым и менее текстовым.

Вместо длинного summary-абзаца нужно:

- сократить основной текст;
- оставить 1 короткий executive-summary абзац;
- усилить KPI-блок карточками;
- добавить отдельный визуальный акцент на `ABC`.

### 4.2 Новый блок `ABC-клиенты в 1С`

Это новое отдельное содержание страницы, размещенное сразу после `Обзора`.

Секция должна включать:

- короткий заголовок `ABC-клиенты в 1С`;
- короткое объяснение, что реализуется автоматическая сегментация клиентов по валовой прибыли;
- статус по этапам в визуальном виде.

Нужно отразить:

- `Этап 1` завершен;
- `Этап 2` выходит / в работе на текущей неделе;
- `Этап 3` связан с автоматизацией расчета и простановки категории;
- ожидание автоматизации: конец текущей или начало следующей недели.

По ТЗ нужно визуально подчеркнуть, что речь идет не про разовую ручную работу, а про системную механику:

- реквизит ABC в карточке клиента;
- история изменений;
- подсказки и ограничения в документах;
- регламентный расчет.

Но на странице это нельзя превращать в длинное ТЗ. Нужно показать это через компактные карточки или этапный прогресс.

### 4.3 Результаты недели по отделу

Из отчета `13–17 апреля` на страницу нужно вынести не перечень задач, а визуально считываемую сводку.

Обязательные метрики/факты:

- `43` задачи всего;
- `23` выполнено;
- `15` в работе;
- `3` заблокировано;
- `2` запланировано;
- база 1С: `230 Гб -> 140 Гб`;
- `400–500` дублей контактов обработано;
- `2000` старых сделок обработано по ортодонтии;
- KPI для сборщиков/упаковщиков завершен;
- QR-код складского задания уже тестировался на складе.

Нужно не перечислять их сплошным текстом, а разложить в карточки, короткие статусы и/или мини-плашки.

### 4.4 Блок `Склад`

В текущей версии по складу есть текстовый roadmap. Его нужно усилить:

- добавить `компактный Гант` по этапам;
- оставить текущий блок `Нужно решение`;
- сократить лишний текст.

Компактный Гант должен строиться по укрупненным этапам:

- Подготовка;
- Разработка;
- Тестирование;
- Доработка;
- Эксплуатация;
- Завершение.

Гант должен быть визуальным, но простым:

- диапазон `апрель -> июнь`;
- горизонтальные полосы по этапам;
- без детализации до каждой строки Excel;
- без попытки повторить настоящий project management tool.

### 4.5 Блок `Сайт`

Сайт остается в странице, но текст должен стать короче.

Нужно сохранить:

- старый сайт: критичные ошибки закрыты;
- новый сайт: запрошены счет и лицензия, переданы референсы;
- учебный центр: протестированы оплата и передача в Bitrix24.

Секция должна по-прежнему содержать:

- CTA-кнопку на Google Sheet;
- быстрые ссылки на ресурсы.

Но визуально она должна быть легче и компактнее, чем сейчас.

### 4.6 Блок `Инциденты`

Текущий блок в целом рабочий, но страница должна стать еще более визуальной.

Допускается:

- оставить основной график;
- усилить карточками рядом;
- не увеличивать объем текста в этом разделе.

Главное — не перегрузить страницу вторым аналитическим отчетом.

## 5. Изменение структуры страницы

Целевая структура после правки:

1. `Обзор`
2. `ABC-клиенты в 1С`
3. `Результаты недели`
4. `Инциденты`
5. `Склад`
6. `Сайт`
7. `Решения`

При необходимости секция `Фокус недели` может быть переработана или поглощена новыми более сильными визуальными блоками.

Если текущий блок `Фокус недели` будет мешать визуальной ясности, его можно убрать как самостоятельный раздел и встроить его смысл в новые карточки.

## 6. Визуальное направление

Стиль страницы сохраняется:

- светлая stone-палитра;
- крупная типографика;
- мягкие тени;
- rounded-карточки;
- аккуратная sticky-навигация.

Но новая версия должна стать:

- более dashboard-подобной;
- более числовой;
- менее текстовой;
- визуально быстрее считываемой.

Графики и плашки должны выглядеть как часть существующего языка, а не как случайно вставленные виджеты.

## 7. Ограничения

Нельзя:

- перегружать страницу длинными абзацами;
- вставлять весь текст ТЗ по ABC;
- вставлять детальный Гант по отдельным задачам;
- ломать текущую публикацию сложными зависимостями;
- превращать страницу в полноценный PM-дашборд.

Нужно:

- сохранить читабельность на встрече;
- показать бизнес-смысл ABC;
- показать Гант по складу в компактной форме;
- подчеркнуть реальные итоги недели по отделу.

## 8. Критерии готовности

Изменение считается готовым, если:

- `ABC` стал отдельным заметным разделом страницы;
- на странице появились дополнительные визуальные метрики по итогам недели;
- в блоке `Склад` есть компактный Гант по этапам;
- текста стало меньше, а визуальных опор — больше;
- страница по-прежнему быстро читается на desktop и mobile;
- ключевые результаты недели считываются за 1-2 минуты без чтения длинных абзацев.
