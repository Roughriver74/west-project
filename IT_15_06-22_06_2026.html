Взаиморасчёты — недопроведение документов

Регламент

«Исполнение оперативной очереди задач» — стоит на 23:00, 1 раз/день. Раньше включали каждые 30 минут — были глюки, откатили.

Константа

Статус

Ринат запустил регламент вручную, проверяет — в нём ли проблема. Разбирается дальше.

🟡 #2

Авто-заполнение серии организацией — сломает ФИФО

Никита и Павел сделали авто-заполнение поля «организация» в сериях при создании. Ринат обнаружил в коде: заполнение безоговорочно для всех номенклатур. Это сломает подбор по ФИФО — система будет искать серию, привязанную к организации, которой может не быть. Критично для лекарственных препаратов (анестетики: Мир-Мед-Профи, Вест-Дент, ЛДПК).

✅ Решение принято

Ограничить авто-заполнение только для анестетиков (по виду номенклатуры / галочке «маркируемая продукция»). Для остальных — организация пустая.

Очистка

Уже заполненные серии очистить групповой обработкой. Механизм отключаемый — сейчас работает старый подбор.

🔴

Неправильное сканирование серий — расхождения сумм. Переход на DataMobile

При сканировании на складе берут не ту серию → создаётся лишняя строка в реализации с ценой по прайсу вместо цены заказа. Пример: перчатки в заказе 100₽, по прайсу 120₽ — при сканировании серии №2 вместо №1 разбивается строка и меняется сумма документа. Бухгалтерия массово отправляет документы на аннулирование; один УПД висел с 3 июня по 22 июня.

DataMobile — внедрение

В DataMobile найдена настройка: ТСД пищит при сканировании «не той» серии, не создаёт новую, а требует взять правильную с полки. Никита едет на склад в понедельник; точка приёмки и бенефициар — Евгений Шикунов.
